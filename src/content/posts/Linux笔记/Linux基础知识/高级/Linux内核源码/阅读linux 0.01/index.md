## 阅读技巧

- linux 0.01的阅读 需要懂c语言

- 阅读源码前 应该了解Linux内核源码的分布情况 现代操作系统一般由进程管理 内存管理 文件系统 驱动程序和网络组成 Linux内核源码的各个目录大致与此相对应

- 在阅读方法或顺序上 有纵向与横向之分 所谓纵向就是顺着程序的执行顺序逐步进行 所谓横向 就是按模块进行 它们通常结合在一起进行

- 对于Linux启动的代码可以顺着Linux启动的顺序一步步来阅读 对于像内存管理部分 可以单独拿出来阅读分析 实际上这是一个反复的过程 不可能读一遍就理解

## 源码阅读

**建议用tree命令查看文件结构 很清晰**

- boot 和系统引导程序相关的代码
- fs 存放linux支持的文件系统代码
- include 存放linux核心需要的头文件 比如asm linux sys
- kernel 和系统内核相关的源码
- lib 库文件
- mm 内存管理相关的代码
