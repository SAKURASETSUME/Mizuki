## 什么是DNS

### IP地址

**作用：标记计算机在网络中的地址信息 保证数据不会发送给别人**

**ip地址的分类**

- A类地址：IP地址范围:0.0.0.0-126.255.255.255，A类地址主要分配给超大型网络公司的环境
- B类地址：IP地址范围:128.0.0.0-191.255.255.255，此类地址主要分配给大型局域网公司环境 127是本地回环地址网段 被电脑内部占用
- C类地址：IP地址范围：192.0.0.0-223.255.255.255 C类地址经常被学校、中小企业等环境使用
- D类地址：D类地址属于广播地址
- E类地址：E类属于系统保留地址

### 公网地址和局域网地址的概念

**局域网的ip地址：用于在某一个群体内的数据交互的ip地址段 离开这个群体这个ip就失效了**

**公网的ip地址：全国唯一的标识某个机器的地址**

### DNS的概念

### 访问网站的原理

**访问网页的时候 本质上是从目标服务器获得了一个html文件 再由浏览器进行渲染**

**访问网站的域名 首先浏览器进行了域名解析 pythonav.cn -> 123.206.16.61**
**浏览器直接访问pythonav.cn 其实是被解析成了 123.206.16.61:80/index.html**

**浏览网页的原理就是**

- 通过互联网找到对方的机器
- 从对方的机器中 找出对应的文件 下载到本地
- 通过浏览器进行渲染这样的数据

### 域名和ip的关系

**域名是更容易被人们记住的字符串 ip地址比较隐嗨 难以记住 因此 出现了各大互联网公司 企业 定制了一个dns服务 ，在一个linux服务器上 安装了dns的服务端软件 提供了一个key-value类型的数据库 记住了域名和ip地址的对应关系**

**比如说**

- **阿里 223.5.5.5 223.6.6.6**
- **腾讯 119.29.29.29**
- **114 114.114.114.114**
- **谷歌 8.8.8.8**

**这些dns服务器地址中 存放了大量的域名解析记录**

**比如 www.baidu.com -> 百度的ip地址**

### 查看域名对应的ip地址

```bash
#单独安装nslookup
yum install bind-utils -y
nslookup 域名 [指定dns服务器地址]

ping 域名
```

### Linux的dns配置文件

**linux的本地dns解析文件 /etc/hosts文件中 这个文件是运维人员自由定义域名和ip强制解析关系的**

**/etc/resolv.conf 该文件 填入的是互联网dns服务器的地址**

## dns服务端的配置 以及什么是DNS劫持

/etc/hosts 本地dns解析 优先级高于dns服务器的解析
/etc/resolv.conf dns服务器的配置 在整个互联网提供域名-ip地址的解析

### dns劫持

**正确的访问流程为： 本地电脑访问域名 -> 机器配置了dns地址 通过dns服务器 将域名对应的ip返回给本地电脑 -> 本地电脑正常访问域名对应的页面**

**被dns劫持后 电脑的dns服务器被修改为了恶意的dns服务器地址 当电脑访问域名时 会在恶意的dns服务器把域名解析为ip地址 如果这个恶意的dns服务器将域名的解析成了另外一个ip地址 那么用户访问的就是恶意网站 而不是输入的域名对应的正确的ip地址**
## dns的解析流程

- 在浏览器输入一个域名 操作系统会先检查本地hosts文件的域名解析关系
- 如果hosts文件没有写死的记录 操作系统会去查找本地的dns缓存(ipconfig /displaydns)是否有解析关系
- 如果hosts文件和本地dns缓存都没有解析关系 操作系统会去网络配置中定义的dns服务器去查找域名解析关系 如果找到了 表示该域名已经在公网中注册了如果没找到 表示该域名未注册
